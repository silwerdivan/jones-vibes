## Lessons Learnt (Phase 3)
- **Ambient Stress:** Narrative tension requires a constant "downward pressure" on stats. Simply adding negative events is not enough if the baseline recovery is too generous.
- **Race Conditions in UI:** `setTimeout` calls that trigger UI updates must be tracked and cancelled when modals close to prevent "ghost" popups.
- **AI/Player State Segregation:** The game engine must strictly distinguish between AI and Human thinking phases to prevent UI event leakage.
- **AI UI Silencing:** AI events that require human interaction (like the turn summary) should be bypassed or automated to prevent blocking the game loop for the human player.
