    - Refined the `dashboardSwitched` event listener to ensure persistence is updated correctly.

## Step 8: Secure Player Actions from AI Interference

Fixed the glitch where player actions could be triggered while the AI was taking its turn.

### Changes:
- **src/game/GameState.ts**:
    - Added `isAIThinking` guards to `workShift`, `takeCourse`, `study`, `travel`, and `applyForJob`.
    - Added optional `isAIAction` parameter to these methods to allow the AI to bypass the guard.
    - Updated `handleAIAction` to pass `isAIAction = true` when calling these methods.
- **src/systems/EconomySystem.ts**:
    - Added `isAIThinking` guards and `isAIAction` parameter to `buyItem`, `deposit`, `withdraw`, `takeLoan`, `repayLoan`, and `buyCar`.
- **src/game/GameController.ts**:
    - Implemented a `guard` helper function in `initializeEventSubscriptions` to wrap `EventBus` handlers.
    - Applied the guard to `WORK_SHIFT`, `TRAVEL`, `APPLY_JOB`, `TAKE_COURSE`, and `STUDY` events.
- **src/main.ts**:
    - Implemented a `guard` helper function in Phase 3 (Event Routing).
    - Applied the guard to all `UI_EVENTS` subscriptions (Banking, Shopping, Turns).
