# Project: Cyberpunk Designer Overhaul

## Objective
Iteratively transform the game into a "poverty trap" that leads to "ruthless empowerment" using the MDA framework from @0.designer-1-input.md and @0.designer-2-input.md.

5.  **Focus** on incremental, unambiguous progress.

## Completed Phases Summary
### Phase 1: Cyberware Foundation & UI Reskin
- Established the **Ripperdoc Clinic** as a new central location for permanent character upgrades.
- Implemented the **Cyberware system** allowing for items with permanent `GameCondition` effects (e.g., *Neural Co-Processor* for study yields, *Synthetic Liver* for happiness/sanity ticks).
- **Thematic Reskin:** Renamed core meters to Happiness ➔ **Sanity (🧠)**, Hunger ➔ **Energy Drain (⚡)**, Cash ➔ **Credits (₡)**.
- **System-wide Integration:** Updated HUD, LifeScreen, ActionCards, and all game/test logic to reflect the new cyberpunk terminology and symbols.
- Verified with 275 passing tests.

### Phase 2: The Subscription Economy & Burnout Mechanics
- **Burn Rate System:** Replaced static daily expenses with a dynamic weekly "Burn Rate" that includes base subsistence and Cyberware maintenance fees.
- **Forced Reboot:** Implemented a burnout mechanic where reaching 0 Sanity triggers an emergency medical extraction, resulting in time loss, credits debt, and a multi-week `TRAUMA_REBOOT` debuff.
- **Codebase Terminology Audit:** Performed a complete rename of `happiness` -> `sanity` and `morale` -> `sanity` throughout all models, logic, data, and tests for total consistency.
- **UI Feedback:** Added a pulsing red visual warning for low Sanity (< 20%) on the Life Screen and updated all labels to "Sanity (🧠)".
- Verified with 280 passing tests.

### Phase 3: The Hustle & Narrative Tension
- **Hustle System:** Added a Gig-Board to the Labor Sector with actions like "Donate Blood-Plasma" and "Scrap Metal", providing immediate credits at the cost of sanity, time, and risk of conditions.
- **Ambient Sanity Drain:** Implemented -10 Sanity per cycle to negate base recovery, forcing players to actively manage mental health.
- **UI & Logic Fixes:** Resolved the Labor Sector double-fire popup, Event Modal overlap (priority queueing), and AI turn interference (locking player actions).
- **Event Deck Expansion:** Added stat-based random events and implemented a `STAT` requirement system for event choices.
- Verified with 290 passing tests and 4 passing Playwright E2E tests.

## Lessons Learnt
- **Terminology Debt:** Renaming core concepts mid-development requires a comprehensive sweep of both implementation and test suites to maintain consistency.
- **Permanent Buffs:** Using `GameCondition` with `duration: Infinity` is an effective, decoupled way to implement permanent "implants" without bloating the `Player` class.
- **Vibe Icons:** Visual symbols (emoji/special characters) in the UI significantly enhance tone but require careful handling in test assertions.
- **Visual Urgency:** High-stakes mechanics like "Burnout" and "Burn Rate" are much more effective when paired with urgent UI feedback (e.g., pulsing red gauges).
- **Terminology Consistency:** Decoupling internal code names from UI labels can lead to confusion; a unified "Sanity" rename across the stack proved much cleaner than mixed "Happiness/Morale" logic.
- **Timeout Management:** Functional `setTimeout` calls in the UI must be tracked and cleared upon modal closure to prevent "ghost" UI updates that trigger after the user has left a location.
- **Event Priority:** Random events and location dashboards can trigger simultaneously; a priority system (tracking `activeEvent` in state) is necessary to prevent UI clashes.
- **Stat Requirements:** Implementing flexible stat requirements (Sanity, Time, Credits) for event choices allows for deeper narrative integration and player agency.
- **Test Stability:** E2E tests involving random elements (like `Math.random`) must be mocked at the browser level to ensure determinism and prevent flakes.
