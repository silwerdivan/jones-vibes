# Project: Cyberpunk Designer Overhaul

## Objective
Iteratively transform the game into a "poverty trap" that leads to "ruthless empowerment" using the MDA framework from @0.designer-1-input.md and @0.designer-2-input.md.

## AI Instructions
1.  **Analyze** the current game state.
2.  **Read** the `Completed Phases Summary` and `Lessons Learnt` below.
3.  **Propose** a single core feature and one smaller "tacked-on" thematic change.
4.  **Create** `2. cyberpunk-designer-overhaul-phase-[x].md` with:
    - Clear tasks and checkboxes.
    - **Enforce Depth:** Include explicit reminders for the executor (e.g., instead of "Add Ripperdoc", write "Add Ripperdoc Clinic AND populate it with at least 3 initial Cyberware items in items.ts").
    - **Strict Validation:** Add validation criteria that require zero search results for deprecated terms (e.g., "Validation: Grep for 'omni creds' returns 0").
    - A defined validation/feedback loop for each change.
    - A "Lessons Learnt (Phase [x])" section.
5.  **Focus** on incremental, unambiguous progress.

## Completed Phases Summary
### Phase 1: Cyberware Foundation & UI Reskin
- Established the **Ripperdoc Clinic** as a new central location for permanent character upgrades.
- Implemented the **Cyberware system** allowing for items with permanent `GameCondition` effects (e.g., *Neural Co-Processor* for study yields, *Synthetic Liver* for happiness/sanity ticks).
- **Thematic Reskin:** Renamed core meters to Happiness ➔ **Sanity (🧠)**, Hunger ➔ **Energy Drain (⚡)**, Cash ➔ **Credits (₡)**.
- **System-wide Integration:** Updated HUD, LifeScreen, ActionCards, and all game/test logic to reflect the new cyberpunk terminology and symbols.
- Verified with 275 passing tests.

## Lessons Learnt
- **Terminology Debt:** Renaming core concepts mid-development requires a comprehensive sweep of both implementation and test suites to maintain consistency.
- **Permanent Buffs:** Using `GameCondition` with `duration: Infinity` is an effective, decoupled way to implement permanent "implants" without bloating the `Player` class.
- **Vibe Icons:** Visual symbols (emoji/special characters) in the UI significantly enhance tone but require careful handling in test assertions.
