# Phase 5: Mobile-First UX Recovery and Onboarding Refresh

## Summary
This phase should be a UI/UX correction pass focused on mobile readability and action clarity, not new game mechanics. The goal is to make the core loop usable on a smartphone-sized screen while removing layout noise that feels cluttered in a cyberpunk context.

## Notes
- The `SECTOR` removal is intended to reduce UI noise; location is already implied by the active screen and the tray when moving.
- The high-contrast time-ring is a critical accessibility fix for mobile where small colored rings were prone to color-blindness conflict.
- The onboarding flow should fully replace the current EULA presentation rather than merely reskin it.
- Hustles should be discoverable immediately in Labor Sector, but the hired job remains the dominant primary action when employed.
