## Step 2: The Burnout Mechanic (Forced Reboot)

Implemented a "Forced Reboot" mechanic that triggers when a player's Sanity (happiness) reaches 0. Instead of an immediate Game Over, the player is subjected to an emergency medical extraction.

### Changes:
- **src/models/types.ts**:
    - Added `MAX_ENERGY` to `ConditionEffectType`.
- **src/data/conditions.ts**:
    - Added `TRAUMA_REBOOT` condition: Reduces Max Energy by 20% for 336 hours (2 weeks).
- **src/EventBus.ts**:
    - Added `BURNOUT_TRIGGERED` to `STATE_EVENTS`.
    - Added `clearAll()` method to `EventBus` to ensure test isolation for singleton-based subscriptions.
- **src/game/Player.ts**:
    - Updated `updateHappiness()` to publish `BURNOUT_TRIGGERED` when happiness hits 0.
- **src/game/GameState.ts**:
    - Subscribed to `BURNOUT_TRIGGERED`.
    - Implemented `handleBurnout()`: resets cycle time to 0, deducts 500 ₡ medical fee, applies `TRAUMA_REBOOT` condition, and logs the event.
    - Updated `checkLoseCondition()`:
        - Removed instant Game Over on 0 happiness.
        - Updated Energy Drain (hunger) failure to check against dynamic `MAX_ENERGY` (default 100).
- **tests/LoseCondition.test.ts**:
    - Updated tests to verify that 0 Sanity triggers Reboot instead of Game Over.
    - Added `EventBus.clearAll()` to `beforeEach`.
- **tests/GameState.test.ts**:
    - Added a new test suite for "Burnout Logic" covering reboot triggers, medical fees, and the `TRAUMA_REBOOT` effect on max energy.
    - Added `EventBus.clearAll()` to `beforeEach`.

### Verification:
- All 280 tests passed (`npm test`).
- Verified that burnout correctly applies conditions and deducts credits via unit tests.
